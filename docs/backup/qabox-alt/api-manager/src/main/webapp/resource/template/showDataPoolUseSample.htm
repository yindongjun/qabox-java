1、管理数据池

要使用数据池，需要依次分别创建 数据池 -> 数据池变量 -> 数据池类别，数据池类别中可以设置当前类别中每个变量对应的值。
通过接口URL或者测试场景可以批量从远端接口获取对应的数据并更新到数据池中，前提是远端接口返回的数据必须为JSON格式且符合指定要求。

2、使用数据池数据

目前数据池数据需要配合测试集使用，在测试集的运行时设置中配置指定的数据池（可配置一个）和数据池类别（可配置同一个数据池下的多个），在测试集执行过程中，同一个测试场景会根据数据池类别的个数生成多个测试场景，并体现在测试报告中;
使用${__数据池变量名}来使用数据池变量，用法同全局变量和测试集公共变量。

3、关于变量数据的使用优先级

在测试报文中使用变量数据时，会根据以下优先级来替换变量数据(从左至右优先级依次降低)：

组合场景内上下文变量 > 测试集公共变量 > 数据池变量 > 全局变量
